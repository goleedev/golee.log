# 💁🏻 웹 접근성이란

웹 개발에서의 접근성은 사람들의 능력이 어느정도 제한되어 있더라도 **가능한 많은 사람들이 웹사이트를 사용 할 수 있도록 하는 것** 입니다.

다시 말해, 사용자의 하드웨어, 소프트웨어, 언어, 문화, 위치 또는 신체적 혹은 정신적 능력이 무엇이든지 간에 **웹은 근본적으로 모든 사람들을 위해 일하도록 설계**되어 있는데,

웹이 이 목표를 달성할 때 **다양한 범위의 청각, 움직임, 시각, 인지 능력을 가진 사람들에게 접근**할 수 있습니다.

---

# 🗺️ 웹 콘텐츠 접근성 지침 (WCAG)

사용자를 위해 웹 접근성을 높이는 `WCAG의 지침`이 있습니다.

1. 모든 텍스트가 아닌 **콘텐츠에 대체 텍스트를 사람들이 원하는 인쇄, 점자, 음성, 기호 또는 간단 언어 등과 같은 형태로 제공**해야 한다.
2. **UX는 운용 가능** 해야한다

- **키보드로 모든 기능 사용 가능**해야 한다
- 사용자가 탐색하고, 콘텐츠를 찾고 그들이 **어디에 위치하고 있는지를 알 수 있도록 도와주는 방법을 제공** 해야 한다

3. 정보와 UX 모두 **어렵지 않고 이해 가능** 해야한다
4. 현재와 미래 기기들 다양하게 해석될 수 있도록 모두 **호환성을 극대화** 해야한다

이처럼 모든 세대, 모든 환경, 모든 신체적 조건, 모든 시간 등 많은 것들을 고려해야지 웹 접근성을 높여 개발할 수 있습니다.

따라서, **모든 인류를 위해 개발할 준비가 되었는지, 웹 접근성을 높이기 위해 개발자로서 뭘 할수 있을 지** 고찰해 보아야 합니다.

---

# 🧑🏻‍💻 웹 접근성 by 개발자

웹 접근성을 높이기 위하여 `개발자가 할 수 있는 것`은 다음과 같습니다.

## 웹 표준을 따라야 합니다.

일관적인 환경을 제공하기 위해 웹 표준에 따라야 하는데, 웹 표준화의 중심에는 `시멘틱 웹`이 있습니다.

이는 **기계가 이해 할 수 있는 리소스를 만들어서 처리하게 끔 하는 웹 개념**이며,

기계가 이해 할 수 있어야 모든 사람이 이해하게 끔 다시 재처리가 가능하기 때문에 웹 표준에 꼭 필요한 개념입니다.

시멘틱 웹은 `HTML5`의 핵심 개념이기도 하기에 웹 개발자가 `HTML5`를 정확히 이해하고 사용해야 웹 접근성을 높일 수 있습니다.

`HTML5`에는 시멘틱 웹을 실현하게하는 많은 태그들이 추가 되었는데,

이를 적절하게 사용해야 **기계가 이 리소스를 적절하게 읽어서 다시 다양한 웹 사용자에게 필요한 방법으로 노출시켜 웹 접근성을 높일 수 있습니다**.

## 다양한 환경에서 다양한 사용자를 가정한 테스트를 해야 합니다.

예를 들어, `VoiceOver`과 같은 기능 또는 **다양한 화면과 기기에서 웹 어플리케이션을 실행 시키면서 불편함이 없는지 확인**해야 합니다.

## 인지적 측면에서 아래와 같은 부분을 고려합니다.

- 사진 비디오 오디오 등 **콘텐츠가 적절히 대체되고 있는 지** 확인합니다.
- 비디오나 오디오에 자막이나 **수화가 제공 되는 지** 확인합니다.

## 운용적 측면에서 아래와 같은 부분을 고려합니다.

- **키보드만으로 조작이 가능한지** 확인합니다.
- **키보드 외외에도 입력이 되는지** 확인합니다.

## 이해적 측면에서 아래와 같은 부분을 고려합니다.

- **오류가 사용자에게 잘 설명 되고 있는지** 확인합니다.
- 사용자 **사용 방향이 예측되는 방법대로 실행 되는지** 확인합니다.
- **사용자가 쓰는 언어로 제대로 전달 되는지** 확인합니다.
