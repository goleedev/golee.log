# 💁🏻 웹 통신의 흐름

![Web Transfer]/images/web-transfer2.png)

1. 사용자가 브라우저에 **도메인 이름을 입력**합니다.
2. **DNS 서버에서 사용자가 입력한 `Domain Name`을 검색**하고, **매핑되는 IP 주소**를 찾습니다. 그리고 사용자가 입력한 **URL 정보와 함께 리턴**합니다.
3. IP 주소는 **HTTP 프로토콜을 이용해서 HTTP 요청 메세지**를 생성합니다.
4. 생성된 HTTP 요청 메세지는 **TCP 프로토콜을 사용해서 인터넷을 거쳐 해당 IP 주소의 컴퓨터 (서버)로 전송**됩니다.
5. 서버는 클라이언트의 **요청을 승인하고, 응답 메세지를 전송**합니다.
6. 도착한 HTTP 응답 메세지는 HTTP 프로토콜을 사용하여 **웹 페이지 데이터로 변환되고, 웹 브라우저의 출력**에 의해 사용자가 볼 수 있습니다.

# 🔍 DNS 서버의 주소 찾기

## DHCP (Dynamic Host Configuration Protocol)

![DHCP]/images/dhcp.png)

호스트의 IP 주소와 TCP/IP 설정을 **클라이언트에 의해 자동으로 제공하는 응용 계층 프로토콜**입니다. 이떄 사용자는 `DHCP 서버`에서 **자신의 IP 주소, 가장 가까운 라우터의 IP 주소, 가장 가까운 DNS 서버의 주소**를 받습니다.

## ARP (Address Resolution Protocol)

네트워크 상에서 **IP 주소를 물리적 네트워크 주소로 바인딩시키기 위해 사용하는 프로토콜**이며, DHCP로 얻은 라우터의 **IP 주소를 MAC 주소로 변환**합니다.

# 🎯 DNS 서버에서 IP 정보 수신하기

> DNS 서버는 계층화 구조를 이루는데, 최상단 계층인 가장 뒷쪽 (.com, .kr 등등)을 담당하는 DNS 서버는 전세계에 13개만 존재합니다.

![DNS]/images/dns.png)

1. DNS 서버에 **`DNS Query (www.example.com)`를 전송**합니다.

2. DNS 서버는 루트 네임 서버에 **`DNS Query`를 질의**하고, 루트 네임 서버는 `.com의 IP 주소`를 반환합니다.

3. **`.com 네임 서버`에 DNS 쿼리를 질의**하고, `.com 네임 서버`는 `example.com의 IP 주소`를 반환합니다.

4. **`example.com 네임 서버`에 DNS 쿼리를 질의**하고, `www.example.com의 IP 주소`를 반환합니다.

# 🌐 웹 서버 접속하기

위와 같은 과정을 통해 서버의 IP 주소를 알아냈습니다. 이제 웹 서버 접속하는 과정이 필요한데요.

1. `HTTP Request`를 위해 `TCP socket`을 개방하고 연결합니다.

![Web Server]/images/web-server.png)

2. TCP 연결에 성공하면, `HTTP Request`가 `TCP socket`을 통해 보내집니다.

3. 웹 페이지의 정보가 응답으로 돌아옵니다.
